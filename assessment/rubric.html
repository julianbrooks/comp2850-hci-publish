COMP2850 • Human-Computer Interaction

COMP2850 HCI: Combined Assessment Rubric

Academic Year: 2025-26 Assessment Type: Coursework (Pass/Fail with 0-100 grading) Pass Threshold: 40/100 marks Marking Time: 5 minutes per portfolio

Quick Reference: 5-Minute Marking Decision Tree

Step 1: Files Present? (30 seconds)

  • submission-template.md (completed)
  • metrics.csv (or pasted in template)
  • evidence/ folder (screenshots, pilot notes)

If NO → FAIL (0-39) - Major gap

Step 2: Minimum Evidence? (1 minute)

  • metrics.csv has n=2+ participants (check session IDs)
  • findings-table.csv has 3+ findings with data sources
  • implementation-diffs.md shows 1-3 code changes
  • verification.csv has 20 regression checks

If NO → FAIL (0-39) - Major gap

Step 3: Fail Triggers? (1 minute)

  • Data too perfect? (all success, no errors, unrealistic times)
  • No pilot notes? (pilots didn’t happen)
  • PII violations? (names/emails in screenshots, identifiable session IDs)
  • Metrics contradict claims? (e.g., says n=4 but CSV shows n=2)

If YES → FAIL (0-39) - Fabrication or severe violation

Step 4: Evidence Chains? (1.5 minutes)

  • findings-table.csv links to metrics.csv lines OR pilot notes timestamps
  • implementation-diffs.md references findings from table
  • verification.csv Part B shows before/after for fixes

Step 5: WCAG Compliance? (1 minute)

  • verification.csv shows no Level A violations remaining
  • At least 1 WCAG criterion improved (fail → pass)

Step 6: Assign Mark (30 seconds)

Grade Descriptors

70-100: First Class (Excellent)

Characteristics:

  • All evidence chains rigorous and complete
  • WCAG compliance thoroughly verified (20/20 checks, no Level A violations)
  • Thoughtful prioritisation with clear rationale
  • Professional documentation (no PII, honest reporting)
  • Limitations acknowledged if metrics didn’t improve
  • Code fixes target root causes, well-integrated

Typical portfolio:

  • n=4 participants, 5+ findings, 3 fixes, complete regression
  • Every finding traceable: data → observation → fix → verification
  • Before/after metrics show measurable improvement
  • Screenshots and SR transcripts support claims
  • Honest analysis of what worked and what didn’t

LO coverage: All 13 LOs demonstrated to high standard

Outcome: PASS

60-69: 2:1 (Very Good)

Characteristics:

  • Strong evidence chains with minor gaps
  • WCAG mostly verified (18-19/20 checks, no critical failures)
  • Good prioritisation with some justification
  • Professional documentation with minor issues
  • Some limitations noted
  • Code fixes work, reasonable approach

Typical portfolio:

  • n=3-4 participants, 4 findings, 2 fixes, mostly complete regression
  • Most findings traceable, 1-2 weak links
  • Before/after metrics present, improvement shown
  • Some supporting evidence (screenshots OR SR transcripts)
  • Mostly honest, minor gaps in reflection

LO coverage: All 13 LOs demonstrated, minor gaps

Outcome: PASS

50-59: 2:2 (Good)

Characteristics:

  • Evidence chains present but incomplete
  • Some WCAG compliance (15-17/20 checks, 1-2 Level A violations)
  • Basic prioritisation, limited justification
  • Adequate documentation, some issues
  • Limitations not well-documented
  • Code fixes work but may be over-engineered

Typical portfolio:

  • n=2-3 participants, 3 findings, 1-2 fixes, partial regression
  • Some findings traceable, weak chains
  • Before/after metrics partial or limited improvement
  • Limited supporting evidence
  • Some honesty, but gaps in analysis

LO coverage: Most LOs demonstrated, some gaps

Outcome: PASS

40-49: Third Class (Adequate)

Characteristics:

  • Weak evidence chains, many gaps
  • Minimal WCAG compliance (<15/20 checks, 2-3 Level A violations)
  • Ad-hoc prioritisation, no clear justification
  • Documentation has issues but meets minimum
  • No limitations acknowledged
  • Code fixes minimal or unclear impact

Typical portfolio:

  • n=2 participants, 3 findings, 1 fix, incomplete regression
  • Few traceable chains, mostly assertions
  • Before/after metrics minimal or absent
  • Little supporting evidence
  • Limited honesty or reflection

LO coverage: Some LOs demonstrated, significant gaps

Outcome: PASS (meets minimum standard)

0-39: Fail

Characteristics:

  • No evidence chains OR fabrication OR severe PII violations OR major gaps
  • Does not meet minimum standard

Automatic fail triggers:

  • Fabrication (0 marks): Data too perfect, no pilot notes, contradictions
  • Severe PII violations (0 marks): Names/emails/student IDs visible
  • Major gaps (0 marks): n<2 OR no metrics.csv OR no regression (<10 checks) OR no redesign

Typical portfolio:

  • Missing files, incomplete data
  • No traceable evidence chains
  • WCAG not addressed
  • Unprofessional presentation
  • Ethical violations

LO coverage: Insufficient demonstration

Outcome: FAIL

Common Patterns by Grade Band

First (70+): What Sets Them Apart

Evidence chains:

  • Can trace complete path for 3+ findings: data → observation → interpretation → fix → verification
  • Data sources specific (e.g., “metrics.csv L47-49 + P2 notes 14:23”)
  • Before/after metrics show clear improvement (or honest analysis of why not)

WCAG compliance:

  • 20/20 regression checks complete
  • Multiple test methods (axe + SR + manual keyboard)
  • No Level A violations, most Level AA pass

Professional practice:

  • Screenshots cropped/blurred (no PII)
  • Session IDs anonymous (P1_xxxx format)
  • Honest limitations (e.g., “small sample size, n=4 not statistically robust”)
  • No-JS parity maintained

Code quality:

  • Fixes target root causes (e.g., “added role=‘alert’ to error span”)
  • Minimal changes, well-integrated
  • Clear “what/why/impact” for each fix

2:1 (60-69): Strong with Minor Gaps

Evidence chains:

  • Can trace 2 complete chains, 1-2 weak links
  • Data sources mostly specific
  • Before/after metrics present, improvement shown

WCAG compliance:

  • 18-19/20 checks complete
  • 2 test methods (axe + manual)
  • No critical Level A violations

Professional practice:

  • No PII violations
  • Some limitations noted
  • No-JS mostly functional

Code quality:

  • Fixes work, reasonable approach
  • Some rationale provided

2:2 (50-59): Adequate with Issues

Evidence chains:

  • Can trace 1 complete chain
  • Data sources vague (e.g., “pilot observations”)
  • Before/after metrics partial

WCAG compliance:

  • 15-17/20 checks complete
  • 1 test method (axe only)
  • 1-2 Level A violations

Professional practice:

  • Minor PII issues (correctable)
  • Limitations not documented
  • No-JS partial

Code quality:

  • Fixes work but over-engineered
  • Limited rationale

3rd (40-49): Minimum Standard

Evidence chains:

  • Few complete chains, mostly assertions
  • Data sources absent or weak
  • Before/after metrics minimal

WCAG compliance:

  • <15/20 checks complete
  • No systematic testing
  • 2-3 Level A violations

Professional practice:

  • Documentation issues
  • No reflection
  • No-JS functional but undocumented

Code quality:

  • Minimal fixes, unclear impact

Fail (0-39): Below Minimum

Automatic triggers:

  • Fabrication (data too perfect, no notes, contradictions)
  • Severe PII violations (names visible, identifiable IDs)
  • Major gaps (n<2, no metrics, no regression, no redesign)

Or insufficient evidence:

  • No traceable chains
  • No WCAG verification
  • Unprofessional

Marking Workflow (5 Minutes)

Minute 1: Quick checks

  • Files present? CSV opens? Spot-check 2-3 screenshots for PII

Minute 2: Pass criteria

  • metrics.csv: n=2+? Columns correct?
  • findings-table.csv: Links to data sources?
  • verification.csv: 20 checks? WCAG Level A pass?
  • implementation-diffs.md: Code changes shown?

Minute 3: Fail triggers

  • Fabrication? (too perfect, no notes, contradictions)
  • PII violations? (checked in step 1)
  • Major gaps? (n<2, no regression, no redesign)

Minute 4: Quality assessment

  • Evidence chains: Complete (70+), Strong (60-69), Adequate (50-59), Weak (40-49)?
  • WCAG: Thorough (70+), Mostly (60-69), Some (50-59), Minimal (40-49)?
  • Professional: Excellent (70+), Good (60-69), Adequate (50-59), Issues (40-49)?

Minute 5: Assign mark

  • 70-100: Excellent (complete, rigorous, professional)
  • 60-69: Very good (strong with minor gaps)
  • 50-59: Good (adequate with issues)
  • 40-49: Pass (meets minimum)
  • 0-39: Fail (below minimum OR trigger present)

Borderline Cases

39 vs 40 (Fail vs Pass)

40 minimum requires:

  • n=2 participants (verified in metrics.csv)
  • 3 findings with some data sources
  • 1 fix with code diff shown
  • Partial regression (at least 10/20 checks)
  • No fabrication, no severe PII violations

If any missing → 39 or below (fail)

49 vs 50 (3rd vs 2:2)

50 minimum requires:

  • Evidence chains present (can trace 1 complete chain)
  • WCAG addressed (15-17/20 checks)
  • Before/after metrics partial
  • Professional presentation

If evidence chains mostly absent → 49 or below

59 vs 60 (2:2 vs 2:1)

60 minimum requires:

  • Strong evidence chains (can trace 2 complete chains)
  • WCAG mostly verified (18-19/20 checks)
  • Before/after metrics present
  • Professional with minor issues only

If chains weak or WCAG gaps significant → 59 or below

69 vs 70 (2:1 vs First)

70 minimum requires:

  • All evidence chains rigorous and complete (3+ traceable)
  • WCAG thorough (20/20 checks, no Level A violations)
  • Honest reporting of limitations
  • Professional throughout

If not rigorous or gaps present → 69 or below
