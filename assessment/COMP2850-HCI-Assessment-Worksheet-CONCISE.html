Assessment Summary

This coursework combines evaluation and redesign in a single portfolio. You will evaluate through peer pilots (Week 9), analyse data to identify issues (Week 10), redesign by implementing 1-3 fixes (Week 10), and verify improvements through regression testing (Week 10-11).

Assessment format: Pass/Fail (40% threshold) with 0-100 grading for formative feedback.

Code resources (Logger.kt, submission template, examples) available on VLE/Gradescope.

Learning Outcomes

On completion of this assessment you should be able to:

  1. Differentiate people-centred design methodologies and justify method selection
  2. Apply needs-finding techniques (linking tasks to Week 6 job stories)
  3. Analyse ethical implications (consent, PII protection, participant rights)
  4. Evaluate digital systems for accessibility (WCAG 2.2 AA regression testing)
  5. Prototype interactive systems (Ktor, Pebble, HTMX)
  6. Apply iterative design (evaluation ‚Üí redesign ‚Üí verification)
  7. Analyse design constraints (no-JS parity)
  8. Design and execute usability evaluations (task-based pilots, metrics)
  9. Apply inclusive design principles (WCAG compliance)
  10. Critique societal impact of technology (systemic barriers)
  11. Collaborate effectively (peer pilots, peer review)
  12. Communicate HCI work professionally (evidence chains, honest reporting)
  13. Integrate HCI with software engineering (server-side instrumentation)

üî¥ Activity 6: Analyse metrics data (30 mins)

üî¥ Activity 7: Prioritise fixes (20 mins)

üî¥ Activity 9: Regression testing (30 mins)

Academic Integrity Reminders

Allowed: AI tools for code generation (cite in submission), piloting with friends (if they consent), testing same participants for re-pilots (note as limitation), discussing approaches in forum

Not allowed: Fabricating pilot data (automatic fail), sharing complete portfolios, using AI for analysis/findings (must be your thinking), copying code without citation

PII violations: Severe (names/emails/IDs visible) = automatic fail + ethics referral; Minor (browser tabs visible) = warning, require correction

Honest reporting: If fix didn‚Äôt work ‚Üí document honestly, analyse why (valued); messy data ‚Üí report faithfully; n=2 only ‚Üí acceptable minimum, state limitation
